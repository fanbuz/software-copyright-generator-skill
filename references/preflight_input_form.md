# 前置采集表规则

用户提出“生成软著材料”“做一下软著生成”或同类请求时，先输出前置采集表，等待用户填写关键字段。没有这些字段时，不直接进入项目扫描、申请表生成或正式材料生成。

可直接运行：

```bash
python3 scripts/generate_input_form.py
```

上层服务可运行：

```bash
python3 scripts/run_stage.py --manifest job.json --stage preflight
```

该阶段会在 `workdir` 写入：

- `前置采集表.md`
- `前置采集字段.json`

## 字段用途

| 字段组 | 用途 |
| --- | --- |
| 登记信息 | 决定申请表核心字段，并用于检查用户手册、源代码材料、文件名、页眉页脚的一致性。 |
| 仓库与代码 | 决定扫描目录、源码候选范围、前后端顺序、排除目录和最终复核词。 |
| 运行与截图 | 决定是否启动系统、如何获取真实页面截图，以及手册是否保留截图预留。 |
| 模板与输出 | 决定是否套打历史申请表模板，以及本次材料输出位置。 |

## 进入下一阶段的最低信息

至少需要确认：

- 软件全称。
- 版本号。
- 著作权人。
- 开发完成日期或明确留空。
- 发表状态。
- 仓库模式。
- 单体项目的项目根目录，或前后分离项目的前端/后端代码位置。
- 截图方式。

其他字段可以写“待确认”，但进入正式生成前必须完成申请表字段、代码选择、截图方式和 Markdown 草稿门禁。
