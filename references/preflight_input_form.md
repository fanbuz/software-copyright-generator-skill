# 前置采集表规则

用户提出“生成软著材料”“做一下软著生成”或同类请求时，先输出前置采集表，等待用户填写关键字段。没有这些字段时，不直接进入项目扫描、申请表生成或正式材料生成。

可直接运行：

```bash
python3 scripts/generate_input_form.py
```

上层服务可运行：

```bash
python3 scripts/run_stage.py --manifest job.json --stage preflight
```

该阶段会在 `workdir` 写入：

- `前置采集表.md`
- `前置采集字段.json`

## 表单分组

表单必须以 `【请用户输入】` 开头，并按以下分组展开：

| 分组 | 覆盖内容 |
| --- | --- |
| 登记与权属 | 软件全称、简称、版本号、著作权人、权利取得方式、权利范围、开发方式、权属补充说明。 |
| 开发与发表 | 开发完成日期、发表状态、首次发表信息、软件分类、硬件环境、操作系统、运行环境、开发工具、版本历史。 |
| 项目形态与仓库 | 单体、前后分离、多仓库、移动端、小程序、桌面端、其他代码位置、仓库快照、依赖排除口径。 |
| 业务范围与用户手册 | 业务领域、目标用户、用户角色、业务模块、不纳入范围、核心流程、手册章节偏好、术语。 |
| 源码材料 | 源码材料策略、优先模块、排列顺序、页数口径、行数口径、局部抽取、源码排除口径。 |
| 运行与截图 | 前端/后端启动命令、移动端/小程序运行方式、访问地址、登录方式、截图方式、截图页面、截图目录、运行限制。 |
| 模板与输出 | 模板模式、信息采集表模板、已有材料目录、输出目录、输出格式、DOCX 预览方式、其他说明。 |
| 正式资料检查 | 不应出现的名称、清理检查词、一致性重点、交付检查要求。 |

## 场景填写提示

- 单体项目填写项目根目录；前后分离项目分别填写前端和后端代码位置；多仓库项目补充其他代码位置。
- 移动端、小程序、桌面端或插件类项目按实际形态填写对应代码位置；没有的形态留空。
- 已有申请表模板或历史材料时填写模板模式和模板路径；没有模板时使用本 skill 生成草稿。
- 可以本地运行的项目填写启动命令和访问地址；无法运行时选择用户提供截图或暂不截图。
- 源码材料默认使用真实源码并保留正常缩进；如需指定前后端顺序、页数口径或排除目录，应在源码材料字段中说明。

## 进入下一阶段的最低信息

至少需要确认：

- 软件全称。
- 版本号。
- 著作权人。
- 开发完成日期或明确留空。
- 发表状态。
- 仓库模式。
- 单体项目的项目根目录，或前后分离/多仓库项目的对应代码位置。
- 截图方式。

按场景还应尽量补充业务模块、核心流程、源码材料策略、模板模式、输出格式和正式资料检查词。其他字段可以写“待确认”，但进入正式生成前必须完成申请表字段、代码选择、截图方式和 Markdown 草稿门禁。
